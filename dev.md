# Phase 2, Iteration 1: array metadata via zarrs + dtype classification + runtime info

## Context

Phase 1 (complete): extendr scaffolding, store cache, `zarrs_node_exists`,
`zarrs_close_store`, R-side availability probing, CRAN tarball script.
All working — 3 Rust functions, ~600 LOC Rust, ~130 LOC R.

This iteration validates the critical prerequisite for the read path: can we
open an array via zarrs, extract metadata, and classify its data type for R?
This exercises `Array::open`, dtype comparison, shape/chunk introspection,
fill value extraction, and zarr format detection.

---

## Changes

### New Rust modules

**`src/rust/src/array_open.rs`** — Shared `open_array_at_path()` helper.
Opens a zarrs `Array` from the cached store at the given path. Reused by
metadata (this iteration) and retrieve (next iteration).

**`src/rust/src/dtype_dispatch.rs`** — `RTypeFamily` enum and `dtype_family()`
function. Classifies zarrs `DataType` into R-compatible type families (double,
integer, logical). Comparison via `PartialEq` against factory functions
(`data_type::float64()`, etc.). Full retrieve dispatch deferred to next
iteration.

**`src/rust/src/metadata.rs`** — `open_array_metadata()` implementation.
Returns a named R list with shape, chunks, dtype, r_type, fill_value_json,
zarr_format, and order. Chunk shape extracted via `chunk_grid().chunk_shape()`.
Zarr format via `match array.metadata() { V2 => 2, V3 => 3 }`. V2 order
extracted by serializing `ArrayMetadataV2` to JSON.

**`src/rust/src/info.rs`** — `runtime_info()` and `set_codec_concurrent_target()`.
Diagnostics and tuning for zarrs parallelism.

### Modified files

**`src/rust/src/lib.rs`** — Added 4 module declarations, 3 `#[extendr]`
wrapper functions, updated `extendr_module!`. Extracted `compiled_features()`
as `pub(crate)` for reuse by `info.rs`.

**`src/rust/Cargo.toml`** — Changed `extendr-api` to git master (fixes
OBJSXP/S4SXP incompatibility with R 4.5.x). Added `zstd` and `transpose`
to default features (pizzarr's default compressor is ZstdCodec; V2 arrays
use F-order which requires transpose codec).

**`R/extendr-wrappers.R`** — Auto-regenerated by `rextendr::register_extendr()`.
Now has 6 `.Call` wrappers (was 3).

### New test file

**`tests/testthat/test-zarrs-metadata.R`** — 7 tests: V2 integer, V3 float64,
3D array, error on missing path, runtime info fields, codec target validation,
codec target round-trip.

## Issues resolved during implementation

1. **extendr-api 0.8.1 + R 4.5.x OBJSXP error.** R 4.5 renamed `S4SXP` →
   `OBJSXP`. The crates.io 0.8.1 release uses `OBJSXP` unconditionally but
   extendr-ffi generates bindings without it. Fix: use extendr git master
   which has a `use_objsxp` cfg flag set by build.rs based on R version.

2. **FillValue doesn't implement Serialize.** Used `as_ne_bytes()` debug
   format instead of `serde_json::to_string()`.

3. **`chunk_grid().chunk_shape()` takes 1 arg, not 2.** Corrected API call.

4. **`Result` generic arg count.** `extendr_api::prelude::*` imports its own
   `Result` type; used `std::result::Result` for non-extendr returns.

5. **Transpose feature required for F-order V2 arrays.** Added
   `zarrs/transpose` to default features.

6. **zstd codec not compiled.** pizzarr defaults to ZstdCodec; added
   `zarrs/zstd` to default features.
